* "Local" variables
* "Global" variables (Read-from)
* "Global" variables (Written-to)
* "External" variables (Read-from)
* "External" variables (Written-to)
* Files (Read-from)^
* Files (Written-to)^
* Subroutines
* Function CALLs

! The difference between "local, global," and "external" variables is as follows:
   * "Local" variables are SET inside of main or a function and only are referenced inside of that function. They are deleted once
     their purpose is served.
   * "Global" variables are SET wherever and are referenced "outside" of where they were originally defined.
   * "External" variables were defined in the HFANSWER.INI file and then read into memory and made "global" variables.
  
  The difference between "read-from" (R) or "written-to" (W) should be self-explanatory, but to clarify only when a variable is
  labeled as "W'd" does it mean its value is being changed wherever (of course if it's an "external" variable then the value copied from HFANSWER.INI
  is being edited, not the actual value written in the file itself). "Local" variables, given they are created in main or function, have the
  distinct property of having been written to in order to be defined, and thus do not have a R/W distinction.
  
  While "internal" variables might be treated explicitely as integers, "external" variables must be treated as strings to minimize input issues.
  Of course, everything is a string in batch, so SET /A's are just there to make it apparent that A) is carries an integer value, and thus B) you can
  use integer comparison operators on said variables.

^ Only HFSLIP-created text files not extracted from other files or pulled from the SOURCE directory (mostly). If a file is created/appended-to
  in a function, and then "read-from," then it is not included in the "Files (R)" section. Only read files from other "namespaces" are included in
  the "Files (R)" section. Renamed files are not counted either.

! Directories are not mentioned.
! Any external .cmd files that are CALL'd will be listed under the "Function CALLs" section.
! Anything REM'd-out will not be included.
